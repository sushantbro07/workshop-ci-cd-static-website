5-Day Training Schedule

Three 2-hour sessions per day. Each day builds on the last — from cloud foundations all the way to a live deployment.

01
Cloud Foundations & AWS Introduction
IaaS / PaaS / SaaS AWS Console IAM
+
  • Session 1
    Cloud Fundamentals
    • Evolution of computing — Traditional vs Cloud
    • Cloud service models: IaaS, PaaS, SaaS
    • Deployment models: Public, Private, Hybrid
    • Benefits: scalability, elasticity, cost-efficiency
  • Session 2
    Cloud Ecosystem & Providers
    • Overview of major cloud providers
    • Introduction to AWS
    • Real-world cloud use cases
  • Session 3
    AWS Basics & IAM
    • AWS Global Infrastructure — Regions & Availability Zones
    • AWS Management Console walkthrough
    • Identity and Access Management: users, roles, policies
02
Core AWS Services & Database Fundamentals
EC2 S3 RDS DynamoDB
+
  • Session 1
    Compute Services — EC2
    • Introduction to Amazon EC2
    • Instance types, key pairs, security groups
  • Session 2
    Storage Services — S3
    • Amazon S3 overview and storage types
    • Use cases for object storage
    • Hands-on: basic S3 bucket usage
  • Session 3
    Database Fundamentals
    • What is a database?
    • SQL vs NoSQL — differences and use cases
    • Overview of Amazon RDS and DynamoDB
    • Choosing the right database for your workload
03
Networking (VPC) & DevOps Introduction
VPC Subnets DevOps Lifecycle
+
  • Session 1
    Networking Basics
    • Fundamentals: IP addressing and subnets
    • Introduction to Amazon VPC
  • Session 2
    VPC Components & Security
    • Public vs Private subnets
    • Internet Gateway concept
    • Security Groups vs Network ACLs
    • Activity: AWS architecture diagram (VPC + EC2 + subnet)
  • Session 3
    DevOps Fundamentals
    • What is DevOps? Traditional vs DevOps workflow
    • DevOps lifecycle: Plan → Code → Build → Test → Deploy → Monitor
    • Key principles: automation, collaboration, continuous improvement
04
Version Control & CI/CD Pipelines
Git / GitHub CI/CD GitHub Actions
+
  • Session 1
    Version Control — Git & GitHub
    • Introduction to Git and GitHub
    • Basic commands: clone, commit, push
    • Branching and collaboration workflows
  • Session 2
    Modern DevOps Concepts
    • Containerization — basic concept
    • Orchestration — basic concept
  • Session 3
    CI/CD Fundamentals & Hands-on
    • Continuous Integration vs Continuous Deployment
    • Pipeline stages: build → test → deploy
    • Tools overview: GitHub Actions / Jenkins
    • Hands-on: create a simple CI/CD pipeline
05
Monitoring, Cost Optimization & Mini Project
CloudWatch Lambda Live Deployment
+
  • Session 1
    Monitoring & Logging
    • Importance of monitoring in cloud environments
    • Introduction to Amazon CloudWatch
    • Metrics vs Logs
  • Session 2
    Advanced AWS Services & Cost Optimization
    • AWS Lambda — serverless concept
    • Amazon SNS — notifications
    • Amazon CloudFront — CDN basics
    • AWS pay-as-you-go pricing model
    • Billing dashboard, budget alerts & cost optimization tips
  • Session 3
    Mini Project & Presentation
    • Deploy a simple application on AWS using EC2/S3
    • Integrate basic CI/CD pipeline
    • Group work, live demo, and feedback session
Learning Outcomes

What You'll Walk Away With

Practical, job-ready skills across cloud infrastructure, DevOps practices, and AWS services.

☁️

Cloud Architecture

Understand IaaS, PaaS, SaaS models and design basic cloud architectures using VPC, EC2, and subnets on AWS.

🗃️

Storage & Databases

Work with S3 for object storage and choose between SQL (RDS) and NoSQL (DynamoDB) databases for real workloads.

🔐

Security & IAM

Configure IAM users, roles, and policies. Understand Security Groups and Network ACLs for access control.

🔁

CI/CD Pipelines

Build and run automated pipelines using GitHub Actions or Jenkins covering build, test, and deploy stages.

📊

Monitoring & Cost Control

Use CloudWatch for metrics and logs. Set up budget alerts and apply cost optimization best practices.

🚀

Live Deployment

Deploy a real application on AWS with a working CI/CD pipeline and present it to the group on Day 5.
